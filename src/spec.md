# Specification

## Summary
**Goal:** Add a shared backend domain types module with placeholder type aliases for future expanded development.

**Planned changes:**
- Create `backend/types.mo` in the backend canister codebase.
- Define empty public type aliases in `backend/types.mo` for `Issue`, `Task`, and `FeedPost`.

**User-visible outcome:** No user-facing changes; the backend codebase gains a shared types module that compiles successfully.
